# QuizX

QuizX 是一个面向 Android / iOS 的题库学习应用原型，当前仓库已搭好最小可运行的 Expo + React Native + TypeScript 界面。

## 当前已实现

- 首页概览
- SQLite 题库表 / 题目表初始化
- SQLite 答题会话与逐题作答记录表
- SQLite 背诵进度表与背诵会话表
- SQLite 背诵会话题目顺序持久化
- SQLite 错题表
- 题库列表从 SQLite 读取
- 题库详情页与题目浏览
- `答题模式` 最小闭环
- `背诵模式` 最小闭环
- `背诵模式` 第二版：按掌握度优先安排题目
- `错题本` 最小闭环
- 系统文件选择器读取本地 Excel
- 本地 Excel 批量选择与顺序导入
- 微信 Excel 导入
- Excel 行数据统一格式化
- 导入预览页与校验提示
- 文件内重复题检测与自动去重
- 基于题目内容的重复题库识别
- 预览确认后写入 SQLite
- 底部导航切换

当前版本仍未包含以下能力：

- 错题记录与复习算法

## 标准导入格式

当前导入只支持这一种标准 Excel 模板，不再兼容旧版的 `题库名称 / 题目 / 选项A~D / 解析 / 标签` 模板。

标准工作表：

- `判断题`
- `单选题`
- `多选题`
- `填空`

非空工作表的第一行表头必须使用下面这 6 列：

- `题干`
- `选项`
- `答案`
- `难度`
- `题型`
- `试题解析`

标准样式来自当前已验证文件 `/home/charms/Downloads/安全责任制.xls`。

字段说明：

- `题干`：题目正文
- `选项`：单列选项文本，使用 `#` 分隔多个选项
- `答案`：判断题使用 `对/错`，单选题使用 `A`，多选题使用 `AB`、`ABC` 这类字母串
- `难度`：当前先读取并校验表头，暂未入库使用
- `题型`：建议与所在工作表一致，如 `判断题`、`单选题`、`多选题`、`填空`
- `试题解析`：题目解析文本，可为空

`选项` 列示例：

- `对#错`
- `A、明日圆舟#B、源神#C、亡者荣耀`

当前支持的题型固定为：

- `判断`
- `单选`
- `多选`
- `填空`

导入规则：

- 一个 Excel 文件会被当作一个题库导入
- 题库名默认使用 Excel 文件名
- 微信导入与本地导入共用同一套标准模板
- 多个工作表会合并进入同一个导入预览
- 同名题库或同名文件只作为提示，不会直接判定为重复题库
- 文件内重复题会在预览页提示，实际入库时只保留首条
- 与已有题库内容完全一致的文件会在预览页提示并默认跳过
- 非空工作表缺少标准表头时会直接报格式错误
- 预览页存在错误行时，当前版本不会允许直接入库
- 判断题答案支持 `对/错`，同时兼容 `A/B`
- 单选/多选答案使用选项字母

## 本地运行

推荐环境：

- Node.js 20 LTS
- npm 10+

安装依赖：

```bash
npm install
```

启动开发服务器：

```bash
npm run start
```

也可以直接使用：

```bash
npm run android
npm run ios
```

如果你在 Linux 上已经通过 Android Studio 安装了 SDK，但路径位于 `~/Android/Sdk`，当前仓库的 `npm run android` 会自动识别这个目录并补上 `adb` 路径。

## 如何测试这个最小可运行界面

### 方式一：用 Expo Go 测试真机

1. 手机上安装 Expo Go
2. 电脑运行：

```bash
npm run start
```

3. 终端或浏览器打开 Expo DevTools 后，使用手机扫码进入
4. 进入应用后，重点检查这些点：

- 首页是否正常显示标题、概览卡片和题库列表
- 点击 `本地 Excel 导入` 按钮，是否能打开系统文件选择器
- 一次选择多个 Excel 后，是否会按队列逐个进入导入预览页
- 点击 `微信 Excel 导入` 按钮，是否也能打开系统文件选择器并进入预览页
- 微信导入预览页里，来源是否显示为 `微信 Excel`
- 预览页里的题型、选项、答案映射是否正确
- 同名文件是否只显示提示，不会直接当作重复题库拦截
- 与已有题库内容完全一致的文件，是否会提示并默认跳过
- 文件内重复题是否只保留首条入库
- 点击 `确认导入到 SQLite` 后，首页题库数量和题目总数是否更新
- 点击首页题库卡片后，是否能进入题库详情页
- 题库详情页中，是否能按题型筛选浏览题目
- 题目卡片展开后，是否能查看选项、答案和解析
- 点击 `答题模式` 后，是否能看到题库选择页
- 选择题库后，判断题/单选题选择选项时是否会自动判分，无需再点提交
- 多选题和填空题是否仍可手动提交并即时判分
- 中途退出后再次进入同一题库时，是否会提示继续上次练习或重新开始
- 最后一题完成后，是否能看到结果汇总页
- 点击 `背诵模式` 后，是否能看到题库选择页
- 选择题库后，是否会优先出现未掌握或较薄弱的题目
- 题目页是否能看到当前题目的掌握等级和背诵次数
- 选择题是否会先显示选项，再点击 `显示答案`
- 展开答案后，是否可以标记 `记住了 / 有点模糊 / 还不会`
- 背诵中途退出后再次进入同一题库时，是否会提示继续上次进度或重新开始
- 背诵恢复后，题目顺序是否与离开前保持一致
- 背诵最后一题完成后，是否能看到结果汇总页
- 完成一轮含错题的答题后，点击 `错题本` 是否能看到按题库分组的错题列表
- 进入某个题库后，是否可以重做错题
- 在错题本中答对后，题目是否会移出错题本
- 在错题本中答错后，题目是否仍然保留
- 错题本完成一轮后，是否能看到移出数量和保留数量
- 点击底部导航，是否能在 `首页 / 答题 / 背诵 / 错题本` 之间切换

1. 安装 Android Studio，并创建一个 Android 模拟器


4. 验证点与真机测试一致

如果仍提示找不到 Android SDK 或 `adb`，先自查这两个命令：

```bash
echo $ANDROID_HOME
which adb
```

在你当前这台机器上，正确的 SDK 路径应为：

```bash
/home/charms/Android/Sdk
```

如果你想手动设置，也可以先执行：

```bash
export ANDROID_HOME=/home/charms/Android/Sdk
export ANDROID_SDK_ROOT=/home/charms/Android/Sdk
export PATH=$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator:$PATH
```
